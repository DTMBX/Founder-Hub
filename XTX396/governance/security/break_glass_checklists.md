# Break-Glass Checklists — B13-P5

## Pre-Activation Checklist

- [ ] Incident meets activation criteria (see playbook §2)
- [ ] Two authorized individuals available (or single-person exception applies)
- [ ] Justification documented
- [ ] Expected duration estimated
- [ ] Secure communication channel established
- [ ] Independent logging system confirmed operational

## During Activation Checklist

- [ ] Emergency credentials accessed from secure store
- [ ] Minimum necessary permissions used
- [ ] Every action timestamped and documented
- [ ] No audit logs deleted or modified
- [ ] No evidence tampered with
- [ ] Regular status updates provided to stakeholders

## Deactivation Checklist

- [ ] Root cause identified or containment achieved
- [ ] All emergency credentials revoked
- [ ] Emergency credentials rotated
- [ ] Normal access controls re-enabled and verified
- [ ] All affected systems health-checked
- [ ] Backup integrity verified (run restore drill)

## Post-Incident Checklist

- [ ] Break-Glass Incident Report completed (within 24 hours)
- [ ] All actions reviewed by Security Officer
- [ ] Root cause analysis documented
- [ ] Stakeholders briefed
- [ ] Procedures updated if gaps identified
- [ ] Audit trail archived to long-term storage
- [ ] Lessons learned documented

## Quarterly Review Checklist

- [ ] Review all break-glass events in period
- [ ] Verify emergency credential accessibility
- [ ] Test communication channels
- [ ] Update authorized personnel list
- [ ] Conduct tabletop exercise
- [ ] Update playbook if needed
